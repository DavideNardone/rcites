Set up a connection to the species plus database

To set up a connection to the Species+/CITES database, an authentication token is required and will be granted once the user have signed up on the Species+/CITES API website. Below, we assume that the user already possesses a token and consider it to be 8QW6Qgh57sBG2k0gtt (as used on the API documentation see https://api.speciesplus.net/documentation).

The token should actually be passed to the header of all URL requests. To ensure so the user has three options:

  1. set an environment variable SPPPLUS_TOKEN in .Renviron file (preferred);

  2. use sppplus_login(), i.e. interactively set SPPPLUS_TOKEN up for the current R session (meaning a login will be needed for the next session);

  1. use the token argument of the functions, i.e. the token is passed to every function calls.

Retrieve a taxon concept identifier

To efficiently query information from the CITES Species+ database, the user first needs to retrieve the unique identifier of a taxon concept (https://api.speciesplus.net/documentation/v1/taxon_concepts/index.html). To do so, the user should first call sppplus_taxon() and provide the scientific name of the taxon. Let us start by requesting the identifier of the African bush elephant, i.e. Loxodonta africana.

library(citesr)


res1 <- sppplus_taxonconcept(query_taxon = "Loxodonta africana")


res1
#>  $all
#>       id          full_name        author_year    rank name_status
#>  1: 4521 Loxodonta africana (Blumenbach, 1797) SPECIES           A
#>                   updated_at active cites_listing
#>  1: 2018-01-31T10:04:10.239Z   TRUE          I/II
#>  
#>  $synonyms
#>        id          full_name      author_year    rank
#>  1: 37069 Loxodonta cyclotis (Matschie, 1900) SPECIES
#>  
#>  $common_names
#>        name language
#>  1: Olifant       AF
#>  
#>  $higher_taxa
#>      kingdom   phylum    class       order       family
#>  1: Animalia Chordata Mammalia Proboscidea Elephantidae

res1 is a list of data.table object with different information pertaining to the naming of the taxon. If you however only need many details, setting appendix_only to TRUE will return the first data.table, all, for instance:

res2 <- sppplus_taxonconcept(query_taxon = "Loxodonta africana", appendix_only = TRUE)


res2
#>       id          full_name        author_year    rank name_status
#>  1: 4521 Loxodonta africana (Blumenbach, 1797) SPECIES           A
#>                   updated_at active cites_listing
#>  1: 2018-01-31T10:04:10.239Z   TRUE          I/II

For other taxon, there might be more than one identifier, in such a case the user should have a look at the active column.

res3 <- sppplus_taxonconcept(query = "Amazilia versicolor", appendix_only = TRUE)


res3
#>        id           full_name      author_year    rank name_status
#>  1:  3210 Amazilia versicolor (Vieillot, 1818) SPECIES           A
#>  2: 65789 Amazilia versicolor (Vieillot, 1818) SPECIES           S
#>                   updated_at active cites_listing
#>  1: 2015-05-07T15:10:59.252Z   TRUE            II
#>  2: 2016-09-23T15:30:46.749Z  FALSE          NULL

Also if the taxon is not listed a warning message should come yup to assert so:

res <- sppplus_taxonconcept(query = "Homo Sapiens")
#>  Warning in sppplus_taxonconcept(query = "Homo Sapiens"): Taxon not listed.
